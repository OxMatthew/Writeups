Virtual Network Phase 2 — Recon and First Attack
Overview

The goal of this lab was to take the internal network I already built and run through a basic offensive workflow inside it. The focus was on identifying live hosts, enumerating services, selecting a target, and then performing some initial recon to see what was exposed.

Lab Setup

The lab consists of:

Kali Linux (attacker)
Ubuntu (target)
Windows 10 (target)
pfSense (router)

All machines are connected to an internal VirtualBox network:

192.168.50.0/24

pfSense is configured as the gateway at:

192.168.50.1

All traffic from the lab is routed through pfSense.

Host Discovery

I started by identifying which systems were active on the network using:

nmap -sn 192.168.50.0/24

From this I confirmed the following hosts:

192.168.50.1 → pfSense
192.168.50.20 → Ubuntu
192.168.50.30 → Windows
Service Enumeration
Windows

I first scanned the Windows machine:

nmap -sV -sC 192.168.50.30

The only service that appeared was:

Microsoft HTTPAPI httpd 2.0 on port 5357

This did not present much of an attack surface, so I moved on.

Ubuntu

Next I scanned the Ubuntu machine:

nmap -sV -sC 192.168.50.20

This showed:

Port 22 → SSH
Port 3000 → Web server

Because it exposed a web service, I selected Ubuntu as the main target.

Web Enumeration

To gather more information about the web service, I ran:

nikto -h http://192.168.50.20:3000

The scan returned several interesting findings including:

a robots.txt file
a /.psql_history file
references to FTP
a /public directory
additional paths
Discovery of Exposed Files

I started manually checking the findings from Nikto.

The robots.txt file contained a reference to an FTP directory. When I navigated to that location in the browser, I was able to access it without any authentication.

The FTP directory contained multiple files that appeared to be hidden or not intended for public access.

This indicates a misconfiguration where internal or sensitive files are being exposed through the web server.

Analysis

This finding shows how small pieces of information can lead to larger issues:

robots.txt revealed a hidden path
the hidden path exposed an FTP directory
the FTP directory contained files that should not be publicly accessible

This demonstrates a common issue where developers rely on obscurity instead of proper access control.
